# Mamba Update — 2026-07-17

## Local Database Shell

- 选择 SQLite 作为每台 Mac 的本机运行数据库；目前不使用 MySQL，也不修改 Evolution 自己的 PostgreSQL。
- 新增 `campaign-data/mamba.sqlite` 数据库壳，文件属于本机运行资料并由现有 `.gitignore` 排除。
- Schema v1 包含 Device、Sender Account、Customers、Conversations、Messages、Operations、Ownership Changes、Sync Jobs 和 Import Runs。
- `sender_accounts` 强制一个 Device 只有一个真实 Sender Phone，同时同一个 Sender Phone 不能静默属于另一台 Device。
- `ownership_changes` 保存修改前后值与重试状态，为之后按 `operation_id` 精确 Rollback 做准备。
- `sync_jobs` 预留 Notion 双向同步、幂等键、失败重试和错误说明；今天没有启动任何同步 worker。
- Settings 新增 Local Database 卡片，可查看 SQLite driver、Schema、文件大小、Device/Sender 和数据数量。
- “初始化 SQLite 数据库壳”只创建本地空表与约束，不读取 Notion、不发送 WhatsApp、不改变 Customer Desk。
- “Notion → SQLite”按钮暂时禁用，等完整 Dry Run、数量对账和 Device Scope 检查完成后才会开放。

## Database Choice

- SQLite 适合当前“一台 Mac 对应一个 WhatsApp 号码”的运行方式：无需常驻数据库服务、离线可用、备份简单。
- 如果未来需要多台电脑实时编辑同一批客户，再迁移到独立的 Mamba PostgreSQL；不建议把 Mamba 表写进 Evolution 的数据库。

## Smart Campaign Scheduling

- 结束时间留空时改为 `AUTO`：按本批实际会发送的人数与安全发送间隔，自动计算尽早完成时间，不再默认拖到晚上 9 点。
- 手动填写 `21:00` 或其他结束时间时才使用 `FIXED`，把客户平均铺在指定时间窗内，并为最后一位客户的 Part 2 预留时间。
- 排队中的 `AUTO` batch 轮到真正启动时，会按当下时间重新排程；不会沿用已经过期的预览时间，也不会突然密集补发。
- Auto 仍保留至少 120 秒的客户间安全间隔与小幅随机延迟；实际完成时间可能受 WhatsApp API、回复检查和网络速度影响而稍晚。
- Flow 1 与 Flow 2–10 两个发送界面都已更新提示，预览会明确显示「自动尽早完成」或「指定时间窗」。
- 新增排程测试，并完成全部 30 个 `test_*.mjs` 测试文件；没有执行 LIVE 发送或写入 Notion 客户资料。

## Closable Batch / Flow Progress

- 每张 Batch / Flow 进度卡新增独立的 `× 关闭` 按钮，不再只能把整个进度区暂时收起。
- 关闭状态保存在本机；刷新或重新打开 Mamba 后不会自动出现，只有点击顶部“恢复已关闭”才会回来。
- 运行中的 Flow 也可以关闭显示，但会先明确提醒：关闭卡片不会停止 WhatsApp 发送或 Notion 收尾。
- 红色“停止”按钮保持独立，避免把“关闭画面”误操作成“停止 Campaign”。
- 关闭不会删除 Campaign 运行记录、客户资料或 Notion 数据。
