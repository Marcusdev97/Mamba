## 今日一句话总结

Mamba 从几个能用但仍需人工盯着的页面，进一步变成较安全的工作系统：发送集中在一个入口、回复服务持续运行、Notion 上传进度可见，而且未处理客户会持续留在 Today Queue，不会无声消失。

## 从什么变成什么

| 范围 | 修改前 | 修改后 |
|---|---|---|
| Campaign 入口 | Flow 1 与 Flow 2-10 看起来像两个独立工具 | 合并为一个 Campaign Center，用 tabs 切换 Flow 1 与 Flow 2-10 |
| Next Flow 安全 | Notion 推进失败时，客户可能还留在旧 Flow | 成功发送后由 Campaign Run Service 推进 Flow，并用测试覆盖 Flow 1-10 |
| Notion 上传反馈 | 上传在后台进行，不知道几时完成 | 显示持续进度层，区分上传中、完成、部分失败和失败 |
| 回复服务 | 关闭页面后追踪可能停止，也可能启动重复服务 | 回复服务由 Mamba Server 管理，并防止同一台电脑重复运行 |
| Telegram 去向 | 部分客户提醒可能发到系统台 | 客户提醒优先发到 Telegram Inbox，旧 Chat ID 只作为 fallback |
| Follow-Up 总数 | STOP 客户混在 Follow-Up 总数里 | Active Follow-Up 排除 STOP 与已完成客户；STOP 仍有独立安全统计 |
| Today Queue | 只有 `Follow Up At = 今天` 才显示，没有日期的工作可能消失 | Today 包含今天到期、已过期和仍需处理但没有日期的客户 |
| 未完成工作 | 客户可能隔天不再显示 | 没有完成动作的客户每天继续留在 Today |
| Agent 动作 | Follow-Up 必须手动选择日期 | `Follow Up`、`Call`、`Send Price` 没选日期时自动排明天 10:00 |
| 完成或拒绝 | 离开队列的规则不统一 | `Done` 完成任务；`STOP` 停止序列、清除日期并进入 STOP 名单 |
| 新客户回复 | Reply 与下一次 Follow-Up 时间分开处理 | 可执行的新回复会建立或刷新 Follow-Up；明确 STOP/终止类回复不会再建立任务 |

### 2. Notion 同步进度

- Campaign 完成后上传 Notion 时显示持续 Loading Overlay。
- 用户刷新页面后，Server 仍保留这次 Run 的上传状态。
- UI 可以区分：
  - 上传中；
  - 已完成；
  - 已完成但部分客户失败；
  - 上传失败。

### 3. 回复服务与 Telegram Routing

- 回复追踪改为由 Mamba Server 持续管理，不再完全依赖某个 Tracker 页面保持打开。
- 增加 Reply Service Manager 测试，避免意外启动重复 worker。
- 确认 Telegram `409 terminated by other getUpdates` 的含义：相同 Bot Token 正被另一台电脑或另一个 process 轮询。
- 客户提醒优先使用 Hub Bot + Inbox Chat ID。
- System Logs 与 Customer Inbox 被视为两个不同的 Telegram 目的地。

### 4. 滚动式 Follow-Up Queue

Follow-Up Desk 现在是每天做工的队列，不只是查看报告的页面。

- 需要处理但未安排日期的客户会出现在 Today Queue。
- Overdue 客户继续留在 Today，同时带有过期标记。
- Hot 是优先级信号，不会把客户从 Today 隐藏。
- STOP 与已完成客户不会出现在 Active Follow-Up。
- 新的可执行回复会马上进入工作队列。
- Agent 完成动作时安排下一次任务：
  - 有填写日期：使用指定日期；
  - 没填写日期：排到明天 10:00；
  - Done：关闭任务；
  - STOP：停止 Sequence，并清除未来 Follow-Up。

安全决定：如果 Mamba 无法确认 Agent 已经完成跟进，客户会继续留在 Today。宁愿多提醒一次，也不要静悄悄漏掉客户。

## Today Queue 现在怎样运行

1. 客户发来一个需要处理的回复。
2. Mamba 保存回复，并把客户放进 Today Queue。
3. Agent 在 Follow-Up Desk 处理客户。
4. Agent 指定未来日期时，使用该日期。
5. Agent 没指定日期时，自动排到明天 10:00。
6. Agent 今天没有处理，客户明天仍在 Today。
7. 客户说 STOP 或 Agent 按 STOP，后续所有 Flow 都会被拦截。
8. 工作真正结束后，Agent 按 Done。

## 当前的重要限制

如果 Agent 直接从手机 WhatsApp 手动回复，却没有在 Mamba 记录动作，Mamba 暂时无法百分百证明这次 Follow-Up 已完成。因此客户仍会留在 Today。这是目前刻意采用的安全行为。

未来可以比较 WhatsApp 最后的 Outbound 与 Inbound Message，自动建议 `Handled Today`；但在逻辑证明可靠之前，不应该静默关闭任务。
