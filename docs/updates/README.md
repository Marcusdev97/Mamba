## 每日日志格式

每一篇应该记录：

1. 修改前系统怎样运行。
2. 发现了什么问题。
3. 从什么改成什么。
4. 怎样测试和验证。
5. 哪些已经 Commit，哪些还没 Commit。
6. 已知风险与下一步建议。

日志中不要保存 API Key、Token、密码或完整客户电话号码清单。
