# Mamba Cloud Assets

Put new image assets here when you want AI or another computer to sync them to
Cloudflare R2.

Suggested folders:

```text
assets/
  inbox/        temporary drop zone
  templates/    reusable template images
  raw/          original source images
  manifest.json generated by the sync tool
```

The sync tool also uploads the existing WhatsApp campaign images from
`campaign-assets/images/`, so old campaign pictures do not need to be moved.

Current cloud setup:

- Bucket: `mamba-assets`
- Prefix: `mamba-assets`
- Public manifest: `https://pub-3e0df885be9c4a2db26f1cfb0fcf8f3e.r2.dev/mamba-assets/manifest.json`
- Sync command: `cd campaign-app && npm run sync:cloudflare-assets`
- Dry-run command: `cd campaign-app && npm run check:cloudflare-assets`

AI rules:

- Put new public image assets in `assets/inbox/` or `assets/templates/`.
- Keep active WhatsApp campaign images in `campaign-assets/images/`.
- Do not commit generated image files from `assets/inbox/`, `assets/templates/`,
  or `assets/raw/`.
- Do not commit `assets/manifest.json`; it is regenerated after sync.
- Do not commit Cloudflare credentials.

Cloudflare credentials stay in `evolution-pilot/.env`, which is ignored by git.
